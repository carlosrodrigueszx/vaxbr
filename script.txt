Estou construindo um backend leve baseado em arquivos, onde dados em CSV são convertidos para Parquet usando PyArrow, recebem uma camada transacional com Delta Lake (delta-rs) — garantindo ACID, versionamento e controle de schema — e são expostos por uma API em FastAPI; a arquitetura funciona como um mini data lakehouse, priorizando leitura eficiente e simplicidade, com uso de compressão (preferencialmente ZSTD) para reduzir I/O e melhorar desempenho, sendo ideal para cenários analíticos ou read-heavy, mas não para cargas intensas de escrita ou alta concorrência.
